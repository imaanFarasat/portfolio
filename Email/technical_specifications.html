Technical Specifications

Pre-Send Quality Assurance Checklist

Before sending any email campaign, I verify these technical specifications to ensure optimal deliverability, rendering, and performance across all email clients and devices.

Code Quality & Structure

✓
HTML ValidationHIGH
Validate HTML code through W3C Validator to ensure proper structure and compatibility.
  • No unclosed tags or syntax errors
  • Proper DOCTYPE declaration
  • Valid HTML5 structure
  • Semantic HTML where possible
✓
Table-Based LayoutHIGH
Use table-based layout for maximum email client compatibility, especially for Outlook.
  • Tables for structure and layout
  • Nested tables for complex designs
  • Proper cellpadding and cellspacing
  • Role="presentation" for layout tables
✓
Inline CSSHIGH
All CSS styles must be inline for maximum email client support.
  • No external stylesheets
  • All styles in style attributes
  • Media queries in <style> tag (limited support)
  • Fallback styles for unsupported properties

File Size & Performance

✓
File Size OptimizationHIGH
Keep HTML file size under 100KB for optimal loading and deliverability.
  • Target: < 100KB total file size
  • Minimize whitespace and comments
  • Remove unused code
  • Optimize HTML structure
✓
Image OptimizationHIGH
Optimize all images for web to reduce load time and improve performance.
  • Compress images (JPG/PNG)
  • Appropriate image dimensions
  • Use WebP with fallbacks (if supported)
  • Lazy loading where applicable
  • Alt text for all images

Responsive Design

✓
Mobile ResponsivenessHIGH
Ensure emails render correctly on mobile devices with responsive design techniques.
  • Viewport meta tag included
  • Media queries for mobile breakpoints
  • Minimum width: 320px support
  • Touch-friendly button sizes (44x44px minimum)
  • Readable font sizes (14px+)
  • Single column layout on mobile
✓
Cross-Device TestingMEDIUM
Test email rendering across different devices and screen sizes.
  • Mobile (320px - 480px)
  • Tablet (481px - 768px)
  • Desktop (769px+)
  • Test on actual devices when possible

Email Client Compatibility

✓
Client-Specific TestingHIGH
Test emails in major email clients to ensure proper rendering.
  • Gmail (Web, iOS, Android)
  • Outlook (2016, 2019, 365, Web)
  • Apple Mail (macOS, iOS)
  • Yahoo Mail
  • Thunderbird
  • Known rendering issues documented
✓
Outlook-Specific FixesMEDIUM
Apply Outlook-specific workarounds for known rendering issues.
  • Conditional comments for Outlook
  • MSO-specific styles
  • VML for background images (if needed)
  • Avoid unsupported CSS properties

Accessibility & Standards

✓
Alt Text & AccessibilityHIGH
Ensure emails are accessible to all users, including those using screen readers.
  • Descriptive alt text on all images
  • Color contrast ratios (WCAG AA: 4.5:1)
  • Readable font sizes (minimum 14px)
  • Descriptive link text (not "click here")
  • Text readable without images
✓
CAN-SPAM ComplianceHIGH
Ensure emails comply with CAN-SPAM Act requirements.
  • Unsubscribe link included and visible
  • Physical mailing address
  • Accurate sender information
  • Clear identification as advertisement (if applicable)

Deliverability & Best Practices

✓
Spam Score CheckMEDIUM
Review email content to avoid spam trigger words and improve deliverability.
  • Avoid excessive use of spam trigger words
  • Proper text-to-image ratio
  • No excessive capitalization
  • Proper link formatting
✓
Plain Text VersionMEDIUM
Provide plain text version for email clients that don't support HTML.
  • Plain text version available
  • Same content as HTML version
  • Properly formatted links
